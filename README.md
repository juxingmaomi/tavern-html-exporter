# 聊天记录 HTML 导出器

适用于 SillyTavern + 酒馆助手的聊天记录导出脚本。

当前稳定版本：`v1.5`

## v1.5 脚本前端静态化

- 识别依赖 JavaScript 解析隐藏数据的 `tx-pack` / `SCENE APPENDICES` 前端。
- 将其转换为原生 `<details>` 静态分区，`SNAPSHOT` 默认展开，其余内容可直接点击展开。
- 移除原始隐藏数据和失效开关，避免导出后只剩一条空标题。
- 放宽消息 iframe 自动高度上限，避免长正文把末尾前端界面藏在内部滚动条下。

## v1.4 性能修复

- 移除导出弹窗的实时背景模糊，降低滚动和合成开销。
- 批量下载按钮只保留文件名和 Blob URL，不再长期引用所有巨大 HTML 字符串。
- 重新打开或关闭导出弹窗时立即撤销旧 Blob URL，并清空下载节点释放内存。

## v1.3 修复

- 识别酒馆助手前端代码块中的完整 HTML，并直接重建为禁止脚本的静态 iframe。
- 即使酒馆助手因渲染深度或懒加载没有创建子 iframe，也能保存 HTML/CSS 界面。
- 自动移除快照里的隐藏源码块，并递归调整嵌套静态 iframe 高度。

## v1.2 修复

- 直接从酒馆主页面的消息 DOM 读取已完成渲染的 `.mes_text`，不再完全依赖酒馆助手的跨窗口查询。
- 修复导出结果全部显示为“酒馆格式”、无法保存酒馆助手前端渲染快照的问题。
- 保留 v1.1 的 `.mes_text` CSS 作用域修复。

## v1.1 修复

- 修复酒馆正则生成的 `.mes_text ...` 作用域样式在导出 HTML 中无法命中的问题。
- 静态快照 iframe 和普通消息容器现在都会保留酒馆的 `.mes_text` 上下文。
- 增加浏览器回归测试，直接验证快照与标准格式化内容的计算样式。

## v1.0 是全新重写版

旧版 `v0.30`、`v0.31` 标签会继续保留，但 `v1.0` 不再沿用旧版渲染核心。

- 每条当前聊天消息只调用一次酒馆显示格式化，避免正则重复执行。
- 优先保存酒馆页面中已经显示的复杂界面，并转成禁止脚本的静态快照。
- 保留 iframe 文档里的 CSS，不保存失效的 `blob:` 地址。
- 移除脚本、事件属性和 `javascript:` 链接。
- 自定义 CSS 通过 `textContent` 写入，无法闭合 `<style>` 注入脚本。
- 支持导入本地 `.json`、`.jsonl` 和 `.txt` 聊天文件。
- 大聊天按指定消息数拆成多个 HTML，避免单文件假分页造成内存暴涨。

## 安装

在酒馆助手中导入：

`聊天记录HTML导出器-GitHub入口壳.json`

入口壳默认加载：

```text
https://gcore.jsdelivr.net/gh/juxingmaomi/tavern-html-exporter@v1.5/index.js
```

刷新酒馆页面后，点击酒馆助手按钮栏中的“HTML导出”。如果按钮栏 API 暂时不可用，右下角会出现备用按钮。

## 更新版本

以后发布 `v1.6`、`v1.7` 时，只需打开入口壳脚本并修改：

```js
const VERSION = 'v1.5';
```

例如以后更新到：

```js
const VERSION = 'v1.6';
```

保存并刷新酒馆页面即可。旧标签不会被覆盖，可以随时把版本号改回旧版。

## 导出模式

### 当前聊天

- 使用酒馆助手公开 API 读取当前聊天。
- 使用 `formatAsDisplayedMessage` 一次性完成宏、酒馆正则和 Markdown 格式化。
- 当前页面已经渲染的复杂前端界面会优先保存为无脚本静态快照。
- 未显示在 DOM 中的楼层会回退到标准酒馆格式。

### 本地 JSON / JSONL

- 不伪造当前聊天的楼层号、角色上下文或正则深度。
- 使用安全文本模式和用户设置的自定义规则生成 HTML。
- 适合导出未打开的聊天文件，但不会假装拥有当前酒馆页面的完整动态界面状态。

## 自定义规则

支持：

- `$0`、`$&`、`{{match}}`：完整匹配
- `$1`、`$2`：数字捕获组
- `$<name>`、`${name}`：命名捕获组
- `/pattern/flags` 格式或分开填写 pattern / flags

自定义规则启用时，该楼层使用标准酒馆格式化，不直接复制现成 DOM 快照。

## 安全限制

- v1.5 不导出或执行聊天内容里的 JavaScript。
- 依赖脚本运行的播放器、游戏或交互组件只保存其静态可见内容。
- 外部图片、字体和音频不会自动转成内嵌文件；离线时可能无法访问这些外部资源。
